##  Sobre o Projeto

<p align="justify"> &emsp;&emsp; Este projeto tem como objetivo documentar as contribuições para Software Livre Tainacan, que trata-se de uma ferramenta flexível e poderosa para WordPress, que permite o gerenciamento e a publicação de coleções digitais com a mesma facilidade de publicação de posts em blogs, mas mantendo todos os requisitos de uma plataforma profissional para repositórios.<br> <br> </p>
  
<p align="justify"> &emsp;&emsp; O objetivo e contruibuir para um software ja avançado para aplicar as tecnicas e ferramentas aprendidas no disciplina de GCES - Gerencia de Configuração e Evolução de Software </p>

##  Sobre o Tainacan

<p align="justify"> &emsp;&emsp; Tainacan é uma solução brasileira, gratuita e de código aberto para gestão e publicação de acervos digitais. Ele transforma seu site WordPress em um repositório dinâmico e personalizável, ideal para instituições culturais, projetos de pesquisa e coleções digitais de qualquer tipo.</p>

<p align="justify"> &emsp;&emsp; Seu objetivo principal é ajudar instituições culturais a organizarem, publicarem e difundirem suas coleções. Ele também pode ser usado para construção de repositórios individuais ou coletivos, para qualquer finalidade, e opera de forma integrada à plataforma WordPress.</p>

## Executando localmente

Siga as instruções abaixo para executar a documentação localmente:

1. Clone o repositório:

```bash
git clone git@github.com:Mayara-tech/GCES_TAINACAN.git
```

2. Entre na pasta do projeto:

```bash
cd GCES_TAINACAN
```

3. Crie o ambiente virtual e instale as dependências:

```bash
make install
```

4. (Opcional) Gere o site para validar se está tudo certo:

```bash
make build
```

5. Execute o servidor MkDocs:

```bash
make serve
```

> Importante: se a pasta `.venv` ainda não existir, execute `make install` antes de `make serve`.

6. Acesse `http://127.0.0.1:8000` no seu navegador para visualizar a documentação localmente.
