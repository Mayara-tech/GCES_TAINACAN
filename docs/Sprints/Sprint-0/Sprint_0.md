# Sprint 0 – Planejamento

A **Sprint 0** tem como objetivo preparar o ambiente e alinhar o entendimento da equipe sobre o projeto, suas tecnologias e funcionamento. Nesta etapa, não há foco em desenvolvimento de funcionalidades, mas sim na criação da base necessária para o início das próximas sprints.

---

## Objetivos da Sprint

- Compreender o propósito do Tainacan  
- Estudar sua arquitetura e principais funcionalidades  
- Identificar as tecnologias utilizadas no projeto  
- Preparar o ambiente de desenvolvimento  
- Organizar o repositório e a documentação inicial  

---

## Atividades Realizadas

### 🔹 Estudo do Sistema

- Análise da documentação oficial  
- Levantamento das principais funcionalidades  
- Entendimento da arquitetura (backend, frontend e API)  
- Identificação dos repositórios do projeto  

---

### 🔹 Levantamento Tecnológico

- Linguagens utilizadas:
  - PHP (backend)
  - JavaScript (Vue.js e React)
- Banco de dados:
  - MySQL / MariaDB
- Plataforma base:
  - WordPress  

---

### 🔹 Preparação do Ambiente

- Instalação das ferramentas necessárias  
- Configuração do ambiente local  
- Execução inicial do sistema (quando aplicável)  

---

### 🔹 Estruturação do Projeto

- Criação do repositório no GitHub  
- Organização inicial das pastas  
- Definição da estrutura de documentação  

---

### 🔹 Publicação da Documentação

- Configuração do MkDocs  
- Criação das páginas iniciais  
- Publicação do projeto no GitHub Pages  

---

## Entregas da Sprint

Ao final da Sprint 0, foram entregues:

- Ambiente preparado para desenvolvimento  
- Documentação inicial do sistema  
- GitHub Pages configurado e publicado  
- Compreensão da arquitetura e funcionamento do Tainacan
